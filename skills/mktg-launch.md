# mktg-launch - Lancement produit VibeCraft

Transforme "j'ai un nouveau produit" en plan exécutable en 30 minutes.

---

## Contexte

Ce skill charge automatiquement le fichier `VIBECRAFT-CONTEXT.md` à la racine du package pour personnaliser ses outputs (ICP, produits, ton, lead magnets).

Si `VIBECRAFT-CONTEXT.md` est absent ou vide : demander à l'utilisateur le contexte produits avant de générer l'output.

## Structure lancement standard (J-7 à J+7)

### Phase teasing (J-7 à J-1)
- J-7 : Post "Je construis quelque chose" - problème sans nommer la solution
- J-3 : Post "Presque prêt" - screenshot ou démo partielle
- J-1 : Post "Demain c'est le jour" - lien d'attente / waitlist si applicable

### Jour J
- Matin : Post principal LinkedIn (le plus complet, avec contexte complet)
- Midi : Post X court avec lien direct
- Reddit : Post dans les subreddits pertinents
- Email liste : Annonce à la liste existante
- Gumroad : Page live avec code promo J0 si applicable

### Phase amplification (J+1 à J+7)
- J+1 : Post "behind the scenes" - comment c'est construit
- J+3 : Premier retour / premier client si possible
- J+7 : Bilan chiffré public (transparence solo founder)

## Règles de comportement

- Toujours demander la date de lancement et le canal principal si non précisés
- Pré-écrire le contenu directement - pas juste des consignes
- Adapter le ton selon le produit (produit technique = plus précis, playbook = plus accessible)
- Signaler si le timing est trop court pour un lancement solide (< 3 jours)
- Proposer un code promo lancement si applicable (ex: LAUNCH20 = 20% J0 uniquement)
- Un lancement à la fois - ne pas superposer deux lancements à moins de 2 semaines
